# 高校虚拟仿真实验教学综合平台 (Clinical Nexus) PRD V3.0 (定稿版) ## 1. 产品概述 Clinical Nexus 是一款专为高校（重点面向医学方向）设计的虚拟仿真实验教学平台。它整合了 Web 3D、高保真 VR 及本地客户端实验资源，通过闭环的“教、学、练、考、评”体系，解决医学实验中高损耗、高风险、难复现的问题。 ## 2. 核心价值主张 - **赋能教师**：支持自主建课、任务发布、报告批改及深度学情分析。 - **沉浸学习**：多模态资源（视频、PDF、3D 模拟）的一体化学习体验。 - **数据闭环**：全过程记录学生实验行为，提供多维度的学情看板。 - **兼容性**：支持多种实验格式，配备环境预检及平台启动器。 ## 3. 用户角色与功能地图 ### 3.1 未登录访客 (Visitor) - **品牌感知**：通过门户首页了解实验室概况、专家团队（关于我们）。 - **信息获取**：查看新闻动态、规章制度。 - **资源预览**：浏览实验大厅、课程中心、资源中心的分类列表及详情，但无法启动实验。 - **身份转化**：通过登录/注册入口进入正式学习环境。 ### 3.2 学生端 (Student) - **学习管理**：个人工作台查看待办任务、最近学习及学分统计。 - **课程学习**：参与“章-节-知识点”结构的课程，完成视频、讲义及实验任务。 - **实验中心**：在实验大厅检索并启动所属学科的仿真实验。 - **报告系统**：在线撰写实验报告，查看教师批改反馈及成绩排名。 - **档案查询**：查看个人学情轨迹与综合成绩单。 ### 3.3 教师端 (Teacher) - **教学工作台**：班级活跃度监控、待办任务（报告批阅）快捷入口。 - **课程建设**：可视化编辑器管理教学大纲，发布含解锁条件的教学任务。 - **报告批改**：双窗格对比视图，支持评分量表与结构化评语。 - **实验管理**：监控虚拟仿真实验运行状态，管理线下实验室排期与设备。 - **学情看板**：下钻查看班级及个体的实验耗时、错误步骤分布。 ### 3.4 机构负责人 & 管理员 (Admin/Dean) - **宏观驾驶舱**：全校活跃度、并发趋势、硬件利用率实时大屏。 - **组织治理**：学校-学院-专业-班级的树状维护，SSO 对接。 - **系统运维**：审计日志、API 密钥管理、CMS 内容发布。 ## 4. 核心业务流程 1. **教师建课**：创建课程 -> 编排大纲 -> 挂载资源 -> 发布任务。 2. **学生学习**：进入课程 -> 完成前置视频 -> 启动仿真实验 -> 提交实验报告。 3. **评价反馈**：教师批改报告 -> 学生查看成绩 -> 系统更新学情画像。 ## 5. 关键非功能需求 - **导航规范**：顶部主导航 + 左侧树形侧边栏，支持角色化菜单定制。 - **环境预检**：启动实验前自动检测显卡、驱动及启动器状态。 - **多端兼容**：支持 PC 浏览器、VR 头显及本地 .exe 回传。
